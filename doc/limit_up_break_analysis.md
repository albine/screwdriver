# 涨停开板封单衰减模式分析

## 背景

LimitUpBreakSellStrategy 使用 300ms 滑动窗口检测封单流出。当窗口内流出量（撤单+成交）超过当前封单量的 40% 时触发卖出信号。

本文分析策略触发后的封单量衰减模式，验证以下假设：
- 封单量衰减速度是否普遍呈现"先快后慢"特征？
- 该特征能否用于区分"真开板"和"回封"？
- 能否通过提高阈值来过滤回封误触发？

**T+0 定义**: 当前 40% 阈值策略触发的时间点。T+100ms 表示触发后 100ms。封单量百分比以 T+0 时刻的封单量为基准。

## 核心发现

### 1. "先快后慢"模式是普遍现象

所有触发的封板段都呈现前 100ms 快速衰减、之后速度放缓的特征。这是因为：
- 第一批流出是大单集中撤单/成交（主力资金先跑）
- 后续流出是散户小单逐步跟撤
- 同时有新买单不断补充

### 2. 回封 vs 真开板的区分特征

| 特征 | 真开板 | 回封 |
|------|--------|------|
| T+100ms 占比 | < 50% | > 60% |
| T+300ms 趋势 | 持续下降 | 停滞或反弹 |
| T+1000ms 占比 | < 30% | > 40% |
| 衰减后期 | 加速（T+800 有二次下跌） | 停滞或回弹 |

**关键判据**: T+300ms 时的走向。如果封单量仍在下降，大概率真开板；如果已经稳住，大概率回封。

### 3. 40% 阈值的问题

当前 40% 阈值导致的误触发案例：
- #2 002082.SZ: 触发后封单量稳定在 69%，15 秒后才真正开板
- #3 600989.SH: 触发后封单量反弹到 113%，是回封而非开板
- #4 600989.SH: 触发后封单量几乎不动 (95%)，是典型误触发
- #6 600989.SH: 提前 433 秒触发，严重误触发
- #10 600227.SH: 触发后封单量稳定在 56%

在 12 个封板段中，策略触发了 11 次，其中 8 次是"开板后回封"，仅 3 次是"最终开板"。**误触发率约 73%**。

### 4. 提高阈值的效果预估

如果将阈值从 40% 提高到 60%：
- #2 (T+100=70%): 不会在 T+0 触发，需要更多流出才触发 → 可能不触发（回封）✓
- #3 (从不低于 59%): 不会触发 ✓
- #4 (T+100=95%): 不会触发 ✓
- #10 (稳定在 56%): 不会触发 ✓

但也可能延迟真开板的触发时间，需要在更多数据上验证。

## 后续建议

1. **提高阈值到 60%** 作为初步优化，可过滤大部分回封误触发
2. **引入二次确认机制**: 在 40% 触发后，等待 200-300ms 观察是否继续衰减（即观察 T+300ms 时占比是否 < 触发时），再决定是否真正发出卖出信号
3. **在更多交易日数据上验证**: 当前仅分析了 3 个交易日的 4 只股票，样本量有限
4. **封单量绝对值过滤**: 600989.SH 段 [4] 和 [6] 的触发时封单量极低（2-4 万股），可设置最低封单量门槛避免噪音触发
